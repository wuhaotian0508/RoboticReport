\begin{abstract}
We study style-aware adaptation for MoConVQ, a physics-based language-conditioned humanoid motion generator. The original proposal planned to fine-tune MoConVQ's language-conditioning transformer on HumanML3D style motions while freezing the motion representation backbone. During implementation, we found that the available HumanML3D parquet export provides processed 263-D motion features rather than MoConVQ-compatible BVH files or discrete motion-token labels. We therefore implement a parameter-efficient LoRA distillation alternative: style captions are filtered from HumanML3D, the pretrained MoConVQ generator produces pseudo token targets, and only low-rank LoRA updates are trained while the pretrained generator remains frozen. This yields a reproducible style-adaptation pipeline with 200 distilled examples, 1.97M trainable parameters out of 195.59M total parameters, measured training loss, generated BVH outputs, rendered qualitative comparisons, and BVH-derived style proxy metrics. After a conservative two-epoch low-learning-rate continuation, LoRA improves 18 of 22 pre-defined proxy comparisons on five style prompts. We do not report FID or R-Precision because the required official evaluator bridge is not available in this workspace.
\end{abstract}

\section{Introduction}
Natural language is an intuitive interface for humanoid motion generation in animation, simulation, and human--robot interaction. MoConVQ combines discrete motion representations with physics-based control, making it a strong baseline for generating BVH humanoid motions from text prompts \cite{yao2024moconvq}. However, general text-to-motion models can be weak at fine-grained style control, such as slow deliberate movement, tai chi-like motion, graceful dance, or heavy tired walking.

Our proposal targeted style-aware fine-tuning under limited compute: keep the motion representation, codebook, decoder, and controller frozen, and adapt only the language-conditioned generator. This remains the organizing principle of the final implementation.

\section{Data and Format Constraint}
HumanML3D is widely used for text-driven human motion generation \cite{guo2022humanml3d}. We curated a style subset using keyword groups for controlled, graceful, dance, energetic, and heavy/awkward motions. Filtering was performed independently inside train, validation, and test splits to avoid leakage.

The available local dataset is the Hugging Face parquet export, which contains processed 263-D motion features and captions. MoConVQ fine-tuning expects its own discrete motion-token targets, generated by its VQ representation. Directly converting the available HumanML3D features into MoConVQ token labels would require a reliable skeleton, coordinate, frame-rate, and representation bridge that is not present in the release. To avoid fabricating supervision, we use pseudo-token distillation.

\section{Method}
The baseline pipeline maps a prompt to text features, predicts a sequence of discrete motion codes, decodes those codes into motion latents, and tracks them with the MoConVQ controller to produce BVH output.

For the final adaptation, the pretrained MoConVQ text generator acts as a teacher. Given a style caption, it deterministically samples pseudo token targets. A student model starts from the same pretrained weights, but all pretrained parameters are frozen. We inject LoRA modules into selected linear layers of the text-conditioned temporal transformer and optimize only those low-rank updates. This keeps the implementation close to the proposal's parameter-efficient frozen-backbone design while avoiding unsupported HumanML3D-to-MoConVQ conversion.

\section{Experiments}
All final runs used a fresh conda environment named \texttt{roboticsreport\_lora} with Python 3.8 and PyTorch 2.4.1+cu118. Baseline reproduction used the official MoConVQ pretrained assets and generated ten BVH files for generic and style-related prompts.

The LoRA run used 200 style captions from the small style-training split. We trained for 5 epochs over a cached pseudo-token dataset. LoRA rank was 8, alpha was 16, dropout was 0.05, and learning rate was $10^{-4}$. The run trained 1,966,080 parameters, 1.005\% of the 195,591,680 parameters in the wrapped text generator.

We then selected the epoch-4 LoRA checkpoint for a conservative continuation run: two additional epochs, learning rate $5 \times 10^{-5}$, same rank, same alpha, and the same frozen pretrained backbone. This continuation is reported because it improved the measured BVH proxy score without changing the metric definition.

We also built a validation pseudo-token cache from 40 held-out style captions. Validation loss is used as a diagnostic for overfitting to pseudo labels; BVH proxy metrics are used separately to evaluate whether generated motions move in the expected style direction.

\section{Results}
The baseline generated ten BVH files under \texttt{outputs/baseline/project\_prompts/}. The initial LoRA adapter generated the same ten prompts, and the selected continued-LoRA checkpoint generated deterministic outputs for the five style prompts. Selected baseline and continued-LoRA outputs were rendered as stick-figure previews for qualitative comparison.

\begin{figure*}[t]
\centering
\includegraphics[width=0.62\linewidth]{../../outputs/figures/lora_qualitative_frames.png}
\caption{Qualitative comparison frames for three style prompts. These images are visual evidence only; they are not a substitute for official FID or R-Precision evaluation.}
\end{figure*}

The initial LoRA training loss decreases in epoch mean from 0.3749 to 0.2721. The continuation run further reduces checkpoint-level training loss to 0.0669. This indicates that the adapter learns to reproduce the teacher's pseudo token targets. It does not prove improved HumanML3D motion fidelity, because the supervision comes from the teacher rather than ground-truth MoConVQ tokens.

For generated BVH motions, we compute proxy metrics that correspond to visible style properties: root speed, mean joint velocity, acceleration, jerk, vertical range, arm motion range, rotation amount, and duration. For each style prompt, we pre-define whether lower or higher values are expected. For example, slow and heavy prompts prefer lower speed and jerk, while energetic jumping prefers higher vertical range and speed.

The proxy metrics therefore support a limited claim: LoRA improves most measured style-direction metrics on the tested prompts, especially slow deliberate walking, energetic jumping, and heavy tired walking. It does not improve every metric; tai chi remains the weakest measured prompt and should be judged by future human evaluation.

\section{Limitations}
The main limitation is data compatibility. The available HumanML3D parquet features are not direct labels for MoConVQ's discrete token space. Our distillation method is therefore a practical adaptation experiment, not full supervised fine-tuning on true HumanML3D motion tokens.

We also do not report official FID or R-Precision. Reporting those metrics would require a compatible evaluator that maps generated MoConVQ BVH outputs and HumanML3D test motions into the same feature space. Since that bridge was not implemented and verified, fabricating benchmark values would be misleading. The final proxy result uses a fixed generation seed for reproducibility, but it is still a small prompt-level evaluation rather than a broad benchmark. Future work should add a validated evaluator, convert or retarget HumanML3D motions into MoConVQ-compatible form, run multi-seed comparisons on held-out prompts, and fill the prepared blind human pairwise evaluation sheet.

\section{Conclusion}
We reproduced the MoConVQ baseline, curated a style-focused HumanML3D caption subset, implemented a parameter-efficient LoRA distillation pipeline, trained and continued a 200-example adapter, generated BVH outputs, computed BVH style proxies, and rendered qualitative comparisons. The final result preserves the proposal's frozen-backbone spirit while honestly addressing the discovered HumanML3D/MoConVQ format mismatch.
