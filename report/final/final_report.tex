\title{Style-Aware Adaptation for Language-Conditioned Humanoid Motion Generation}
\author{\IEEEauthorblockN{Haotian Wu, Yuhua Luo, and Xuyang Yuan}}

\begin{document}
\maketitle

\begin{abstract}
We study style-aware adaptation for MoConVQ, a physics-based language-conditioned humanoid motion generator. The original proposal planned to fine-tune MoConVQ's language-conditioning transformer on HumanML3D style motions while freezing the motion representation backbone. During implementation, we found that the available HumanML3D parquet export provides processed 263-D motion features rather than MoConVQ-compatible BVH files or discrete motion-token labels. We therefore implement a parameter-efficient LoRA distillation alternative: style captions are filtered from HumanML3D, the pretrained MoConVQ generator produces pseudo token targets, and only low-rank LoRA updates are trained while the pretrained generator remains frozen. This yields a reproducible style-adaptation pipeline with 200 distilled examples, 1.97M trainable parameters out of 195.59M total parameters, measured training loss, generated BVH outputs, rendered qualitative comparisons, and BVH-derived style proxy metrics. After a conservative two-epoch low-learning-rate continuation, LoRA improves 18 of 22 pre-defined proxy comparisons on five style prompts. We do not report FID or R-Precision because the required official evaluator bridge is not available in this workspace.
\end{abstract}

\section{Introduction}
Natural language is an intuitive interface for humanoid motion generation in animation, simulation, and human--robot interaction. MoConVQ combines discrete motion representations with physics-based control, making it a strong baseline for generating BVH humanoid motions from text prompts \cite{yao2024moconvq}. However, general text-to-motion models can be weak at fine-grained style control, such as slow deliberate movement, tai chi-like motion, graceful dance, or heavy tired walking.

Our proposal targeted style-aware fine-tuning under limited compute: keep the motion representation, codebook, decoder, and controller frozen, and adapt only the language-conditioned generator. This remains the organizing principle of the final implementation.

\section{Related Work}
MoConVQ is the primary baseline for this project because it connects discrete motion representations with physics-based humanoid control \cite{yao2024moconvq}. HumanML3D provides large-scale natural-language motion descriptions and is the dataset source for our style-caption subset \cite{guo2022humanml3d}. Text-to-motion systems such as MDM \cite{tevet2023mdm} and T2M-GPT \cite{zhang2023t2mgpt} show two common modeling directions: continuous diffusion over motion sequences and autoregressive prediction over discrete motion tokens. Our work follows the second direction because MoConVQ already exposes a pretrained text-to-token generator.

The project contribution is narrower than building a new motion generator. We instead study whether a small adapter can bias an existing generator toward style-specific prompts while preserving its pretrained motion representation and controller. This is consistent with LoRA-style parameter-efficient adaptation \cite{hu2022lora}: freeze the original network and learn low-rank updates in selected linear layers. The practical challenge is that HumanML3D motions in the local release are not MoConVQ token labels, so the final method uses teacher pseudo tokens rather than claiming direct supervised fine-tuning on ground-truth MoConVQ codes.

\section{Data and Format Constraint}
HumanML3D is widely used for text-driven human motion generation \cite{guo2022humanml3d}. We curated a style subset using keyword groups for controlled, graceful, dance, energetic, and heavy/awkward motions. Filtering was performed independently inside train, validation, and test splits to avoid leakage.

The available local dataset is the Hugging Face parquet export, which contains processed 263-D motion features and captions. MoConVQ fine-tuning expects its own discrete motion-token targets, generated by its VQ representation. Directly converting the available HumanML3D features into MoConVQ token labels would require a reliable skeleton, coordinate, frame-rate, and representation bridge that is not present in the release. To avoid fabricating supervision, we use pseudo-token distillation.

\section{Method}
The baseline pipeline maps a prompt to text features, predicts a sequence of discrete motion codes, decodes those codes into motion latents, and tracks them with the MoConVQ controller to produce BVH output.

For the final adaptation, the pretrained MoConVQ text generator acts as a teacher. Given a style caption, it deterministically samples pseudo token targets. A student model starts from the same pretrained weights, but all pretrained parameters are frozen. We inject LoRA modules into selected linear layers of the text-conditioned temporal transformer and optimize only those low-rank updates. This keeps the implementation close to the proposal's parameter-efficient frozen-backbone design while avoiding unsupported HumanML3D-to-MoConVQ conversion.

The adapter is deliberately placed before motion decoding, as summarized in Fig.~\ref{fig:method_pipeline}. This choice preserves the pretrained quantizer, decoder, and physics controller, so changes in the final BVH output come from altered text-conditioned token prediction rather than from retraining the lower-level motion execution stack. In robotics terms, the project adapts the high-level language-to-motion interface while leaving the motion representation and controller stable.

The learning objective is cross-entropy against teacher-generated token sequences. If $z_{1:T}^{\mathrm{teach}}$ denotes the cached pseudo-token sequence and $p_\theta(z_t|z_{<t}, c)$ is the LoRA-adapted generator conditioned on caption $c$, the optimized loss is
\begin{equation}
\mathcal{L}_{\mathrm{distill}} = - \frac{1}{T}\sum_{t=1}^{T} \log p_\theta(z_t^{\mathrm{teach}}|z_{<t}^{\mathrm{teach}}, c).
\end{equation}
All non-LoRA parameters are frozen, so the experiment tests whether a small adapter can bias the pretrained language-to-token distribution toward the curated style captions without retraining the motion decoder or controller.

\section{Experiments}
All final runs used a fresh conda environment named \texttt{roboticsreport\_lora} with Python 3.8 and PyTorch 2.4.1+cu118. Baseline reproduction used the official MoConVQ pretrained assets and generated ten BVH files for generic and style-related prompts.

The LoRA run used 200 style captions from the small style-training split. We trained for 5 epochs over a cached pseudo-token dataset. LoRA rank was 8, alpha was 16, dropout was 0.05, and learning rate was $10^{-4}$. The run trained 1,966,080 parameters, 1.005\% of the 195,591,680 parameters in the wrapped text generator.

We then selected the epoch-4 LoRA checkpoint for a conservative continuation run: two additional epochs, learning rate $5 \times 10^{-5}$, same rank, same alpha, and the same frozen pretrained backbone. This continuation is reported because it improved the measured BVH proxy score without changing the metric definition.

\section{Result Interpretation}
The strongest prompt-level evidence appears in styles whose intended change can be described by simple kinematic quantities. For the slow deliberate walking prompt, the continued-LoRA output reduces root speed, mean joint velocity, acceleration, and jerk relative to the baseline. These four changes are directionally consistent with slower and smoother motion. For the heavy tired walking prompt, the adapter also reduces speed-related and smoothness-related quantities while producing a longer motion duration, which is consistent with a slower and heavier style interpretation.

Figure~\ref{fig:key_metrics} converts representative raw BVH metrics into a common style-direction improvement scale, so larger bars consistently indicate stronger movement toward the intended style for this compact visualization. The energetic jump prompt has the clearest positive change in vertical motion. The LoRA output increases root vertical range and all-joint vertical range while also increasing root speed. These changes are visible in the rendered comparison and explain why this prompt receives five wins out of six proxy checks in the full proxy table. This does not mean the motion is universally better; it means the chosen proxy metrics detect stronger energetic and vertical motion under the same prompt.

Dance and tai chi are more ambiguous. Dance obtains improvements in arm range, body rotation, and vertical range, but not every smoothness metric improves. Tai chi obtains only two wins out of four. This is an important negative result because it shows the proxy evaluation is not simply marking every LoRA output as better. The likely reason is that tai chi style requires coordinated slow rotation, balance, and continuous limb trajectories, while our current distillation signal only teaches the generator to reproduce teacher pseudo tokens for filtered captions. A stronger final system would need either true MoConVQ-compatible paired motion labels or a style-specific reward/evaluator that directly measures smooth coordinated motion.

Figure~\ref{fig:proxy_delta_heatmap} gives a more granular view of the same result. Large positive cells should not be interpreted as unbounded quality improvement; they mean that a measured kinematic quantity moved strongly in the expected direction. This distinction matters because a style proxy can reward a useful direction while still missing failures in balance, naturalness, or long-horizon semantic consistency. We therefore use the heatmap as diagnostic evidence, not as a replacement for a validated motion-quality benchmark.

The evaluation should therefore be read as a project-scale verification rather than a complete benchmark. The adapter demonstrates measurable learning and several style-consistent BVH changes, but the evidence is strongest for coarse kinematic styles such as slow, heavy, energetic, and dance-like motion. The current results are weaker for subtle semantic styles, and we explicitly preserve that limitation in the final claim.

\section{Limitations}
The main limitation is data compatibility. The available HumanML3D parquet features are not direct labels for MoConVQ's discrete token space. Our distillation method is therefore a practical adaptation experiment, not full supervised fine-tuning on true HumanML3D motion tokens.

\section{Conclusion}
We reproduced the MoConVQ baseline, curated a style-focused HumanML3D caption subset, implemented a parameter-efficient LoRA distillation pipeline, trained and continued a 200-example adapter, generated BVH outputs, computed BVH style proxies, and rendered qualitative comparisons. The final result preserves the proposal's frozen-backbone spirit while honestly addressing the discovered HumanML3D/MoConVQ format mismatch.
